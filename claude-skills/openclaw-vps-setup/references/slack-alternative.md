# Slack as the Chat Layer — Phase 5 Alternative

This document replaces **Phase 5 (Wire Discord)** and the Discord-specific portions of the 7-step onboarding playbook from `SKILL.md`. Phases 1-4 (provision VPS, install Docker + OpenClaw, configure SOUL/AGENTS/HEARTBEAT, render `openclaw.json`) and Phase 6 (Boot + verify) are unchanged — only the channel wiring differs.

Use this when the user prefers Slack over Discord — typically because their team already lives in Slack and adding a Discord workspace just for agents is friction.

---

## Mental model

Slack and Discord both deliver the same outcome: each agent posts in a dedicated room, watchable in real time. The differences are mostly on the Slack side of the wire:

| Concern | Discord | Slack |
|---|---|---|
| **Tokens per agent** | 1 (bot token) | **2** (bot token `xoxb-…` + app-level token `xapp-…`) |
| **Transport** | Gateway intents (built into bot token) | **Socket Mode** — requires a separate app-level token with the `connections:write` scope |
| **App creation** | Discord Developer Portal → New Application → Bot | Slack API Console → Create New App → **From manifest** (paste OpenClaw's recommended JSON) |
| **Scope count** | 3 (Send Messages, Read History, View Channels) | 16 bot scopes (see manifest below) |
| **Install model** | OAuth2 invite URL per server | "Install to Workspace" once, then `/invite @bot` per channel |
| **Channel ID** | Numeric, copied via Developer Mode | `C…` prefix (public) or `G…` (private), copied from **About → Channel ID** |
| **One identity per…** | One bot per agent (you make N Discord apps) | One Slack app per agent (you make N Slack apps in the same workspace) |
| **Config schema** | `channels.discord.{bot_token_env, default_channel_id}` | `channels.slack.accounts.<id>.{botToken, appToken, channels.default}` — multi-account by default |
| **Bonus surface** | None beyond chat | Slash command `/openclaw`, App Home messages tab for DMs, pins, reactions, file uploads |

Bottom line: Slack is ~3x more setup-clicks per agent, but the manifest-driven flow does most of the work in one paste. Budget ~5 min per agent for Slack vs ~3 min for Discord.

## Phase 5-Slack — Wire Slack

Goal: agent #1's chat I/O lands in a dedicated Slack channel via Socket Mode.

### Step 5.1 — Create the Slack app from the OpenClaw manifest (~2 min)

1. Open https://api.slack.com/apps → **Create New App** → **From manifest**.
2. Pick the user's Slack workspace from the dropdown → **Next**.
3. Paste this JSON. Replace `OpenClaw` (4 places) with the agent's display name — e.g. `RFI Triager`:

```json
{
  "display_information": {
    "name": "OpenClaw",
    "description": "OpenClaw connector for OpenClaw"
  },
  "features": {
    "bot_user": {
      "display_name": "OpenClaw",
      "always_online": false
    },
    "app_home": {
      "messages_tab_enabled": true,
      "messages_tab_read_only_enabled": false
    },
    "slash_commands": [
      {
        "command": "/openclaw",
        "description": "Send a message to OpenClaw",
        "should_escape": false
      }
    ]
  },
  "oauth_config": {
    "scopes": {
      "bot": [
        "chat:write",
        "channels:history",
        "channels:read",
        "groups:history",
        "im:history",
        "mpim:history",
        "users:read",
        "app_mentions:read",
        "reactions:read",
        "reactions:write",
        "pins:read",
        "pins:write",
        "emoji:read",
        "commands",
        "files:read",
        "files:write"
      ]
    }
  },
  "settings": {
    "socket_mode_enabled": true,
    "event_subscriptions": {
      "bot_events": [
        "app_mention",
        "message.channels",
        "message.groups",
        "message.im",
        "message.mpim",
        "reaction_added",
        "reaction_removed",
        "member_joined_channel",
        "member_left_channel",
        "channel_rename",
        "pin_added",
        "pin_removed"
      ]
    }
  }
}
```

4. **Next** → **Create**. One paste gets you: bot user, all 16 scopes, all event subscriptions, the `/openclaw` slash command, and Socket Mode pre-enabled.

**Source of truth**: this manifest is generated by `~/openclaw/src/channels/plugins/onboarding/slack.ts` (`buildSlackManifest()`). If upstream OpenClaw updates the required scopes or events, regenerate from there.

### Step 5.2 — Install to workspace and grab the bot token (~30s)

1. Left sidebar → **Install App** → **Install to <Workspace>** → **Allow**.
2. You land on the OAuth tokens page. Copy the **Bot User OAuth Token** (`xoxb-…`).
3. That token is `SLACK_BOT_TOKEN_AGENT_1` in your env. The OpenClaw Slack plugin also reads a bare `SLACK_BOT_TOKEN` env var for the default account — useful for the first agent.

### Step 5.3 — Generate the app-level token for Socket Mode (~30s)

1. Left sidebar → **Basic Information** → scroll to **App-Level Tokens** → **Generate Token and Scopes**.
2. Token name: `openclaw-socket`. Scope: `connections:write` (the only allowed scope). **Generate**.
3. Copy the token (`xapp-…`). That's `SLACK_APP_TOKEN_AGENT_1`.

Both tokens are now in hand. The manifest already turned Socket Mode on — no additional toggle needed.

### Step 5.4 — Create the channel and invite the bot (~1 min)

In Slack (the user's workspace):

1. Create channel `#<role>-agent` (e.g. `#rfi-triager-agent`). Private if you want it hidden from the rest of the workspace.
2. In that channel: type `/invite @<bot-name>` → enter. **Mandatory** — unlike Discord, Slack bots cannot see messages in a channel they haven't been explicitly invited to, even with workspace-wide install.
3. Click the channel name at the top → **About** tab → scroll to the bottom → copy the **Channel ID** (starts with `C` for public, `G` for private; ~11 chars).

### Step 5.5 — Wire tokens + channel ID into the VPS (~1 min)

Append both tokens to `~/openclaw/.env`:

```bash
ssh openclaw-vps 'cat >> ~/openclaw/.env <<EOF
SLACK_BOT_TOKEN_AGENT_1=xoxb-PASTE-HERE
SLACK_APP_TOKEN_AGENT_1=xapp-PASTE-HERE
EOF
chmod 600 ~/openclaw/.env'
```

Then edit `~/.openclaw/openclaw.json` — replace the Discord `channels` block with the Slack version. Use the **single-account form** for the first agent (simplest):

```json
{
  "agents": {
    "rfi_triager": {
      "model": { "primary": "anthropic/claude-sonnet-4.7" },
      "heartbeat": { "every": "60m" },
      "skills": [],
      "mcp_servers": [],
      "channels": {
        "slack": {
          "botToken": "${SLACK_BOT_TOKEN_AGENT_1}",
          "appToken": "${SLACK_APP_TOKEN_AGENT_1}",
          "channels": { "default": "C0123456789" }
        }
      }
    }
  }
}
```

Or the **multi-account form** if you want to use a single OpenClaw instance to talk to multiple Slack workspaces, or to keep tokens namespaced for clarity once you have several agents:

```json
{
  "channels": {
    "slack": {
      "accounts": {
        "agent_1": {
          "enabled": true,
          "botToken": "${SLACK_BOT_TOKEN_AGENT_1}",
          "appToken": "${SLACK_APP_TOKEN_AGENT_1}",
          "channels": { "default": "C0123456789" }
        }
      },
      "defaultAccount": "agent_1"
    }
  }
}
```

The Slack plugin merges top-level `channels.slack` settings with per-account overrides (see `~/openclaw/src/slack/accounts.ts` → `mergeSlackAccountConfig`), so either form works. Pick the simpler one for a single agent.

✅ **Phase 5-Slack complete = app exists, both tokens stored, channel exists with bot invited, channel ID wired into openclaw.json.**

Proceed to **Phase 6 — Boot + verify** as written in SKILL.md. The only adjustment: in Step 6.3, grep the logs for `slack` instead of `discord`:

```bash
ssh openclaw-vps 'docker logs --tail 50 openclaw-openclaw-gateway-1 2>&1 | grep -iE "slack|socket"'
```

You're looking for a successful Socket Mode connection (no `invalid_auth`, no `app_token_missing`) and an `app_mention` subscription confirmation. Then in the Slack channel, `@<bot-name> hello, introduce yourself.`

## 7-step onboarding playbook — Slack adjustments

For each additional agent, the playbook from SKILL.md applies with these per-step deltas:

| Step | Discord | Slack |
|------|---------|-------|
| 3 | Append `{ channels: { discord: { bot_token_env, default_channel_id } } }` | Append `{ channels: { slack: { botToken, appToken, channels.default } } }` |
| 4 | Add **1** new env var (`DISCORD_BOT_TOKEN_AGENT_N`); create channel; OAuth-invite bot | Add **2** new env vars (`SLACK_BOT_TOKEN_AGENT_N` + `SLACK_APP_TOKEN_AGENT_N`); create channel; `/invite @bot` |
| Slack-only prep before Step 3 | — | Create a **new Slack app from manifest** with a unique display name; install to workspace; generate both tokens. Each agent = its own Slack app. |

Everything else (sync skills, restart gateway, trigger from chat) is unchanged.

## Gotchas (things that don't bite you on Discord)

1. **Both tokens are required.** Bot token alone won't connect — without `xapp-…`, Socket Mode can't open the websocket and the agent never sees events. Logs will show `app_token_missing` or `connections:write missing`.
2. **`connections:write` is the only scope on the app-level token.** Don't add others; Slack will refuse to generate. The app-level token is a separate auth surface from the bot token.
3. **`/invite @bot` per channel is mandatory.** Workspace install does not auto-add the bot to channels. Silent agents almost always mean missing invite.
4. **App display names must be unique within a workspace.** When you make a second Slack app for agent #2 in the same workspace, Slack rejects the duplicate name. Pick distinct display names per agent (e.g. `RFI Triager`, `Submittal Reviewer`, `Comms Drafter`).
5. **DMs require the App Home messages tab.** The manifest above turns this on (`messages_tab_enabled: true`). If you build an app manually without that, DMs silently fail.
6. **Channel ID is from About → Channel ID, not the URL.** The URL contains the ID but also extra path segments. Use the Channel ID field at the bottom of the About panel.
7. **One Slack app per agent.** Don't try to share one app across agents — display name, scopes, and the slash command all live at the app level, so multi-agent in one app means they all look the same and route to the same place.
8. **The `apps/` directory on the VPS is Swift code** for the macOS/iOS clients, not chat plugins. Chat plugins live in `~/openclaw/extensions/<name>/` (TypeScript). Look there if you need to verify behavior, not in `apps/`.

## Troubleshooting checklist

If the agent doesn't respond to `@mention` after Phase 6:

```bash
# 1. Is the gateway up?
ssh openclaw-vps 'docker ps --format "{{.Names}} {{.Status}}" | grep openclaw'

# 2. Did Slack auth succeed?
ssh openclaw-vps 'docker logs openclaw-openclaw-gateway-1 2>&1 | grep -iE "slack.*auth|invalid_auth|app_token_missing" | tail -20'

# 3. Are both tokens present in .env?
ssh openclaw-vps 'grep -E "^SLACK_(BOT|APP)_TOKEN" ~/openclaw/.env | sed "s/=.*/=***SET***/"'

# 4. Is the bot actually in the channel?
# In Slack: channel name → About → Members. The bot user must be listed.

# 5. Does the gateway see the channel ID?
ssh openclaw-vps 'docker logs openclaw-openclaw-gateway-1 2>&1 | grep -i "channel" | tail -20'
```

Common causes:

| Symptom | Likely cause |
|---------|--------------|
| `invalid_auth` on bot token | `xoxb-…` token revoked or workspace uninstalled — reinstall the app |
| `app_token_missing` | `SLACK_APP_TOKEN` not in `.env` or not interpolated into `openclaw.json` |
| Bot online, silent on `@mention` | Bot not invited to channel — run `/invite @<bot>` in Slack |
| Bot online, silent on DM | `messages_tab_enabled` was `false` when app was created — edit App Home settings in Slack API console and toggle on |
| Gateway crashes on boot | `channel.id` malformed (used URL fragment instead of `C…` ID) |
